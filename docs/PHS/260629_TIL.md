### 동시성과 멀티태스킹
- 동시성 : 여러 작업을 짧은 간격으로 교대하며 동시에 실행되는 것처럼 보이게 만드는 방식
- 실제 병렬성과는 다름 > 단일 코어에서도 시분할 방식으로 여러 작업 처리 가능
- I/O 대기 시간을 줄여 CPU와 시스템 자원 활용도를 높이는 데 유리

```java
Runnable task = () -> {
    int localCounter = 0; // 로드 시 각 스레드의 스택에 따로 존재
    localCounter++;
};
```

### 프로세스와 스레드
- 프로세스 : 실행 중인 프로그램 단위
- 스레드 : 프로세스 내부에서 실제 작업을 수행하는 실행 흐름 단위
- 힙 메모리는 공유, 스택 메모리는 스레드별 독립 > 지역 변수는 별도 동기화 없이도 안전

```java
int localCounter = 0; // stack
System.out.println(Thread.currentThread().getName() + " / " + localCounter);
```

### 멀티스레딩과 동기화, 데드락
- 경쟁 상태 : 여러 스레드가 공유 자원에 동시에 접근해 결과가 꼬이거나 데이터가 유실되는 현상
- `synchronized`는 간단한 기본 락, `ReentrantLock`은 `tryLock` 등 세밀한 제어 가능
- 락 해제를 보장하지 않으면 데드락 위험 발생 > `finally`에서 `unlock()` 처리 필요

```java
lock.lock();
try {
    count++;
} finally {
    lock.unlock(); // 예외가 나도 반드시 해제
}
```

### 자바 동시성 유틸리티
- `AtomicInteger` : CAS 기반 원자 연산 제공 > 락 없이도 안전한 증가 처리 가능
- `ConcurrentHashMap` : 버킷 단위로 동시 접근 제어 > 멀티스레드 환경에 적합
- `HashMap`은 스레드 안전하지 않음 > 동시 수정 시 데이터 유실이나 구조 손상 가능

```java
AtomicInteger atomicCount = new AtomicInteger(0);
atomicCount.incrementAndGet(); // CAS 기반 원자적 증가

ConcurrentHashMap<String, Long> concurrentMap = new ConcurrentHashMap<>();
concurrentMap.put("key", System.currentTimeMillis());
```

### 비동기 처리
- 비동기 : 작업을 맡긴 뒤 호출한 쪽은 결과를 기다리며 멈추지 않는 처리 방식
- `CompletableFuture` : 비동기 작업의 결과 가공, 체이닝, 예외 처리를 한 흐름으로 구성 가능
- 기존 `Future`보다 논블로킹 파이프라인 구성에 유리

```java
CompletableFuture.supplyAsync(() -> "Hello!")
        .thenApply(s -> s.toUpperCase()) // 결과 가공
        .exceptionally(e -> "ERROR")
        .thenAccept(System.out::println);
```

### 동시성 <-> 비동기 정리
- 동시성 : 여러 작업을 효율적으로 배치하고 처리하는 실행 관점의 개념
- 비동기 : 호출 시점과 결과 수령 시점을 분리하는 제어 방식
- 둘은 별개 개념 > 비동기가 항상 멀티스레드를 의미하지는 않음

### 함수형 프로그래밍이란?
- 함수형 프로그래밍 : 상태 변경과 부작용을 줄이고, 함수를 값처럼 다루는 프로그래밍 스타일
- 자바에서는 람다식, 함수형 인터페이스, 스트림 API로 이를 지원
- 반복문 중심 명령형 코드보다 "무엇을 할지"를 선언적으로 표현하기 쉬움

### 함수형 인터페이스
- 함수형 인터페이스 : 추상 메서드가 하나만 있는 인터페이스
- `@FunctionalInterface` : 함수형 인터페이스 규칙 위반을 컴파일 단계에서 검증
- 람다식은 함수형 인터페이스 구현을 간결하게 작성하기 위한 문법

```java
@FunctionalInterface
interface CustomPrinter {
    void print(String msg);
}
```

### 람다식 (Lambda Expression)
- 람다식 : 익명 클래스를 더 짧은 함수 형태로 표현한 문법
- 캡처한 지역 변수는 `final` 또는 effectively final 이어야 함
- 동일한 동작의 기존 메서드가 있으면 메서드 참조로 더 간단히 표현 가능

```java
CustomPrinter printer = text -> System.out.println("람다식에서 " + text);
Runnable r = Solution06::runNew; // 메서드 참조
```

### Comparator
- `Comparator` : 두 값을 비교해 정렬 순서를 결정하는 규칙
- 객체의 특정 필드를 기준으로 오름차순, 내림차순 정렬을 유연하게 정의 가능
- 단순 뺄셈 비교보다 `Comparator.comparing` 계열이 가독성과 안정성 면에서 유리

```java
list.sort((e1, e2) -> e1.a() - e2.a()); // a 기준 오름차순
list.sort((e1, e2) -> e2.b() - e1.b()); // b 기준 내림차순
```

### Java Stream 연산과 종류
- 스트림 : 데이터를 선언적으로 가공하는 파이프라인
- 중간 연산 : `filter`, `map`, `sorted` 등 > 스트림을 반환하며 연산 흐름만 구성
- 최종 연산 : `forEach`, `reduce`, `toList` 등 > 실제 실행을 트리거
- 지연 연산 특성상 최종 연산 전까지는 실제 처리가 일어나지 않음

```java
names.stream()
        .filter(name -> name.length() > 3)
        .limit(1) // 필요한 만큼만 처리
        .map(String::toUpperCase)
        .forEach(System.out::println); // 최종 연산
```

### 병렬 스트림
- 병렬 스트림 : 스트림 작업을 여러 스레드로 분할 실행하는 기능
- 계산량이 크고 독립적인 작업에는 유리하지만, 작은 작업이나 공유 상태 수정에는 비효율적일 수 있음
- `forEach(list::add)` 같은 부작용은 지양 > `.toList()`나 `collect(...)`로 안전하게 결과 수집

```java
List<Long> result = LongStream.rangeClosed(1, count)
        .parallel()
        .boxed()
        .toList(); // 공유 리스트에 직접 add 하지 않음
```
