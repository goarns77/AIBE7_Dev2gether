# 🌱 JVM 스터디: 내 자바 프로그램은 어떻게 도는 걸까?

## 🎯 스터디 목표
- 복잡한 JVM 내부 구조를 달달 외우기보다는, 실제 코드가 실행되는 전체적인 흐름을 숲의 관점에서 이해합니다.
- `OutOfMemoryError`나 `StackOverflowError` 등 실무에서 마주칠 수 있는 대표적인 메모리 에러의 발생 원인을 스스로 파악할 수 있는 눈을 기릅니다.

## 📚 주차별 커리큘럼

### [1주차] 자바는 어떻게 실행될까? (그림으로 이해하는 JVM)
- **자바 삼총사 (JVM, JRE, JDK):** 내 컴퓨터에서 자바를 실행하려면 뭐가 필요할까?
- **자바 코드가 실행되는 마법:** `.java` 파일이 `.class` 파일(바이트코드)로 바뀌고 실행되기까지의 간단한 흐름
- **클래스 로더 맛보기:** JVM이 내가 짠 코드를 읽어들이는 과정 아주 쉽게 이해하기

### [2주차] 자바의 메모리 방 구조 (어디에 무엇이 저장될까?)
- **Heap과 Stack의 차이:** 
  - **Heap 영역:** `new` 키워드로 만든 객체들이 모여 사는 넓은 운동장
  - **Stack 영역:** 함수를 호출할 때마다 차곡차곡 쌓이는 개인 책상
- **흔한 에러 구경하기:** 
  - `OutOfMemoryError`: 객체를 너무 많이 만들어서 메모리가 꽉 찼을 때!
  - `StackOverflowError`: 함수가 무한으로 자기 자신을 호출했을 때!

### [3주차] 메모리 청소부, 가비지 컬렉터 (GC) 
- **가비지 컬렉션(GC)이란?:** 아무도 안 쓰는 데이터를 알아서 치워주는 고마운 기능
- **쓰레기인지 어떻게 알까?:** 아무도 가리키지 않는(참조가 끊긴) 떠돌이 객체 찾기
- **청소할 때는 시간이 멈춘다?:** 메모리 청소를 위해 잠시 프로그램이 멈추는 현상(Stop-The-World) 가볍게 이해하기

### [4주차] 메모리 청소 구경하기 & 설정해보기
- **청소부의 세대 교체:** 왜 객체를 '젊은 객체'와 '늙은 객체'로 나누어 따로 청소할까? (Young / Old 영역의 존재 이유)
- **내 프로그램 메모리 눈으로 확인하기:** IntelliJ 프로파일러나 VisualVM 툴로 메모리 그래프 구경해보기
- **자주 쓰는 설정:** 서버를 실행할 때 메모리 크기 정해주기 (`-Xms`, `-Xmx` 옵션이 대체 뭘까?)
